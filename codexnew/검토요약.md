## 1. 게이트③ 서명·승인 체계 (완료)

신청 → **1차 발급** → 업체 현장 TBM → **2차 입회** → **별지(공무)확인** → **작업개시** → **종료신고 → 종료확인**.
단계 뱃지는 `src/lib/work-permit-stage.ts`가 **서명 데이터로 계산**(status 컬럼은 포털 브리지와 공유되어 신뢰 불가).

| 단계 | 트리거 | DB 저장 | 비고 |
|---|---|---|---|
| 신청 | `/work-permit/info`→참여자→서약·각서·교육(`/docs`) | work_permits + participants | 소장 포함 전원 참여자 |
| **1차 발급(일원화)** | 관리자 `PATCH /api/admin/work-permits/[id]` `action:issue` **또는** 포털 `POST /api/bridge/work-permits/[id]/status` `action:APPROVE` | `issuer_signature·approved_by·approved_at` | **B안**: 포털 승인 = 관리자 발급과 동일 1행. **둘 다 손서명 필수**, status 미변경 |
| 업체 현장 TBM | `/work-permit/tbm/[id]` (본인확인 후) | `tbm.photos`, `tbm.confirmations`(작업자 돌려서명) | 사진 + 작업자 서명 |
| **2차 입회** | `action:witness` (1차 후) | `tbm.witness` + `tbm.safetyInstructions` | TBM 미비 시 **경고 모달**, 안전지시사항 필수 |
| **별지 3차 현장확인** | `action:dept_confirm` / `action:dept_proxy` | `dept_confirmations[key]` | 담당부서 매트릭스(§9). **화기·정전(공무) 미확인 시 작업개시 차단** |
| 긴급 대리확인 | `action:dept_proxy` | `mode:EMERGENCY_PROXY`+사유 | **공무 별지 한정·SUPER 전용·사유 필수** |
| **작업개시 승인** | `action:start_work`(서명 없음, 게이트) | `started_by·started_at`, status `APPROVED` | 1차·2차·공무별지 전부 확인돼야 통과 |
| 종료신고 | `action:complete_report` | `completion.workerSignature·reportBy` | 작업자/소장(안전환경 대리입력) |
| 종료확인 | `action:complete_confirm` | `completion.confirmSignature`, status `COMPLETED` | 안전환경 최종 |

- 권한: 안전환경 계열 = `WORKPERMITS_APPROVE`(SUPER 통과) / 공무 별지 정상확인 = `WORKPERMITS_DEPT_CONFIRM`(명시부여) / 긴급대리 = SUPER.
- 서명자 표기: 이메일 → "부서 이름 직책"(`admins` 프로필, `src/lib/work-permit-signer.ts`). 미등록 시 이메일 앞부분.

## 2. 인쇄·출력 (게이트③-5, 완료)

- **인쇄 화면 `/work-permit/print/[id]`** = 회사양식(xlsx)을 **HTML로 재현**(D안). QR로 연결, 7개 구획(TBM·사진·마스터·별지·교육·서약·각서).
- 공개 `GET /api/work-permits/[id]`(**정화됨**: 전화번호·이메일 제거, 서명자 표시명 변환) 재사용 + `qrDataUrl`·`tbmPhotoUrls` 확장.
- **xlsx `GET /api/work-permits/[id]/xlsx`** = 법적 원본(ExcelJS, `src/lib/work-permit-template.ts`). 서명 아래 로그 PNG(@napi-rs/canvas).
- 화면 하단 "전자 확인용 — 공식 서식은 회사양식(.xlsx)" 명시. **양식-인쇄 동기화 규칙**(맨 아래 §) 적용.

## 3. 검색 (관리자 완료 / 외부 현행 상이 — 확인 요망)

- **관리자 목록 `/admin/work-permits`** (완료): `GET /api/admin/work-permits?month=YYYY-MM`. 월 선택기(**◀ 제한 없음 / ▶ 다음 달까지**), 진입 시 **이번 달 자동조회**, 겹침필터(work_start≤월말 AND work_end≥월초), 통합검색(신청번호·업체·작업명·신청인), "서명 미완료만 보기", **10개씩 클라 페이지네이션**.
- **외부 본인조회 `/work-permit/my`** (현행): `POST /api/work-permits/my-list`. **이름·생년월일·연락처 본인확인** + **작업예정일 범위(dateFrom~dateTo) 날짜 검색**. 페이지네이션 없음(전건 표시). issued 건은 [현장 TBM 진행] 버튼.
- ⚠ **불일치**: 사양의 "외부 조회 = 월 선택기·6개월 제한·5개씩"은 **현재 코드에 미반영**. 월 선택기는 관리자 목록에만 존재하고, 외부(`/work-permit/my`)는 여전히 날짜범위 방식·페이지네이션 없음. → 외부 개편이 필요하면 별도 작업 필요.

## 4. 포털 관제판 (완료)

- **포털 `/access`**(`components/access/AccessPageClient.tsx`): "오늘의 작업 관제판" 2섹션 — 🔵 오늘 진행 중(단계칩) / ✅ 오늘 완료(종료확인 시각). **60초 자동갱신, 보기 전용**(조작 버튼 없음).
- 중계 `GET /api/access/today`(로그인 필수) → 브리지 `GET /api/bridge/today`(safety-edu).
- 브리지: **`x-bridge-key` 게이트**, `dynamic=force-dynamic`+`fetchCache=force-no-store`. **개인정보 0** — 응답은 허가번호·업체명·작업종류·단계·(완료)시각뿐. 이름·전화·서명 미반환(서명은 단계 판정용으로만 서버 내부 사용).
- 실패 시 `{available:false}` 반환, throw 없음(페이지 나머지 정상).

## 5. 이전 단계로 되돌리기 (완료 — "반려"와 별개)

- 목적: 서류를 버리지 않고 **마지막 완료 단계만 한 칸 되감기**(예: 2차 취소 → TBM 인원 보완 → 2차 재승인).
- `PATCH /api/admin/work-permits/[id]` `action:rollback`. 상세화면 마지막 단계 줄에 **[↩ 이전 단계로 되돌리기]** 버튼 + 사유 모달 + 되돌리기 이력 접이식.
- 규칙(서버 강제): **한 번에 1칸**(별지 → 2차 입회 → 1차 발급 순), **사유 필수**, **작업개시(started_at)·종료 이후 거부**, **SUPER 전부 / 그 외 본인 서명 단계만**.
- 취소 시 **해당 단계 서명·서명자·시각만 삭제**(다른 단계 무손상, **TBM 사진·작업자 서명 불가침**). 1차 취소 → 승인 대기 복귀(포털 승인대기 목록 재등장).
- **`rollback_logs`(migration 016, JSONB)** 에 `{stage,label,supKey,by,at,reason}` **영구 append**(삭제 불가). 공개 GET은 `rollbackLogs`를 표시명으로 변환해 노출(이메일 차단).
- 서명 삭제는 데이터 삭제라 **xlsx·print에서 자동으로 사라짐**(둘 다 서명 조건부 렌더).

## § 양식-인쇄 동기화 규칙 (유지)

인쇄 화면(`/work-permit/print/[id]`)은 회사양식(xlsx)을 **재현**한 것이므로, 아래 중 하나라도 바꾸면 **print 반영 여부를 확인**하고 보고서에 한 줄 포함: **[양식/서명 변경 → print 반영: 예/불필요]**
- ① 양식 파일(`work-permit-template.xlsx`) 수정(칸 추가/삭제/이동)
- ② 서명 종류·위치·로그 규칙 변경
- ③ xlsx 채움 로직(`work-permit-template.ts`) 수정
- 법적 원본 = xlsx. print는 공개 `GET /api/work-permits/[id]`(정화됨) 재사용, 부족분만 GET 확장.
