## 4. 승인 규칙 (2026-07-07 확정)

> 배경: 종전에는 작업허가 승인이 **안전환경담당 단독 서명**으로 끝나 책임이 담당자에게 집중되는 구조였음. 이를 작업 주관 부서로 이관하여 책임소재를 분산한다.

**원칙: 작업을 요청·관할하는 현장 책임자가 승인하고, 안전환경담당은 검토·발급만 한다. 안전환경 단독 승인 금지.**

| 역할 | 담당 | 책임 성격 |
|---|---|---|
| 신청인 | 작업 수행자 (자체 정비직원 / 생관업체 현장책임자) | 작업 이행 |
| **승인자** | **요청·주관부서 현장 책임자 (차장·대리급)** | **작업 승인 책임** |
| 발급자 | 안전환경담당 | 안전조치 검토·발급 |
| 부서 책임자·관련부서 협조자·입회자 | 기존 서명칸 유지 | 현장 안전조치 확인·협조·입회 |

- **12종 허가서 공통 적용** — 위험도에 따라 승인선이 갈리지 않음(단일 트랙).
- 승인자 칸은 기존 `직책 ___ 성명 ___ (서명)` 그대로 사용 → 차장/대리 직책 자유 기재, **셀 구조 변경 없음**.
- 자체 정비 작업: 신청인 = 정비 작업자, 승인자 = 정비부서 현장 책임자.
- 생관업체(협력업체) 작업: 신청인 = 업체 현장책임자, 승인자 = 관할 사내부서 현장 책임자.
- **공장장(안전보건관리책임자) 승인 단계는 두지 않음** — 요청부서 책임자 승인만으로 책임 분산 목적 달성.

## 5. 결정 사항

- **시트2 디지털 헤더(작업일·승인 서명이미지·QR)는 시트2(대표 허가서)에만 적용** — 확정(2026-07-07). 나머지 허가서 시트(3·4·5·6·7a·7b·7c)에는 미적용, 현행 구조 유지.
- **위험도 뱃지 → 삭제(철회)** — 공장장 승인 단계를 두지 않아 위험도로 승인선이 갈리지 않으므로, 뱃지의 기능적 용도가 사라져 제외(2026-07-07).
- 승인 규칙(§4)은 **12종 허가서 전체 공통** — 승인란은 기존 셀 그대로 사용, 구조 변경 없음.
- 개정 5개 시트 외 7개 시트는 **셀 구조·좌표 변경 없음** (자동입력 시 기존 셀 좌표 그대로 사용).

## 6. 구현 원칙 (2026-07-07 확정)

- **별지 출력 원칙**: 템플릿 원본 12시트는 **절대 불변**. 출력물(발급본)은 종전처럼 **체크된 별지(해당 허가서)만 포함**한다. 미체크 별지를 출력물에서 제외하는 로직은 **유지**(제거하지 않음). ← 앞선 초안의 "삭제 로직 제거" 지시를 이 원칙으로 대체.
- **승인 DB 컬럼**: 기존 `approved_by` / `approved_at`은 **이름·의미 변경 금지** — 안전환경담당의 **발급·검토** 기록으로 유지. 요청부서 현장 책임자 승인은 **신규 `approver_*` 계열**로 분리: `approver_name` / `approver_title` / `approver_signature` / `approval_mode`(SITE|REMOTE) / `approver_signed_at`. migration은 `ADD COLUMN IF NOT EXISTS`.
- **진행 게이트**: migration SQL은 코드 push 전에 **사용자가 Supabase에서 먼저 실행**(실행 안내 필요). 구현은 두 지점에서 중간 보고 — ① 데이터 구조+UI 완료 ② 양식 출력+검증 완료.
